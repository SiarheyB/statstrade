# Запуск сервера из iPhone (iSH)

Готовые скрипты к разделу **«Сервер из iPhone»** корневого
[README](../../README.md). Полное объяснение (туннель, домен, Гид-доступ) —
там; здесь только сами файлы и порядок запуска.

- **`setup.sh`** — разовая настройка: ставит пакеты, скачивает `cloudflared`,
  создаёт `.env` и генерирует секреты, `npm install`, миграции, сборка.
  Идемпотентный — можно запускать повторно.
- **`start.sh`** — запуск/надзор: поднимает приложение и Cloudflare-туннель,
  перезапускает упавший процесс. Если продакшн-сборки нет — стартует в
  dev-режиме. Запускать в `tmux`.

## Порядок (внутри iSH)

```sh
git clone <URL> ~/tradestats && cd ~/tradestats

sh deploy/iphone/setup.sh      # 1-й запуск создаст .env и сгенерирует секреты
nano .env                      # впиши DATABASE_URL (строка подключения из Neon)
sh deploy/iphone/setup.sh      # 2-й запуск: install + миграции + сборка

# туннель Cloudflare — см. корневой README, шаг 6 (cloudflared login / create / route / config.yml)

tmux new -d -s srv "$PWD/deploy/iphone/start.sh"   # запуск сервера в фоне
tmux attach -t srv             # логи (выход из просмотра: Ctrl+B, затем D)
```

После перезагрузки телефона: открыть iSH и снова выполнить строку
`tmux new -d -s srv ...`, затем включить Гид-доступ и «Автоблокировку: Никогда».
